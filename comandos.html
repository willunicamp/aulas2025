Principais Comandos do Git

Uma referência rápida para os comandos mais utilizados no dia a dia com o Git.

git config

Configura valores específicos para o seu ambiente Git. É o primeiro comando que você deve executar após instalar o Git para configurar seu nome de usuário e e-mail.

Exemplos de Uso:

# Configura o nome de usuário globalmente
git config --global user.name "Seu Nome"

# Configura o e-mail globalmente
git config --global user.email "seu.email@exemplo.com"
git init

Inicia um novo repositório Git em um diretório existente. Isso cria uma subpasta oculta .git que contém todos os arquivos necessários para o controle de versão.

Exemplo de Uso:

# Navegue até a pasta do seu projeto e execute
git init
git clone

Cria uma cópia local de um repositório remoto existente. Este comando baixa todo o histórico do projeto e o prepara para trabalho local.

Exemplo de Uso:

# Clona um repositório a partir de uma URL
git clone https://github.com/usuario/repositorio.git
git status

Mostra o estado atual do seu diretório de trabalho e da área de preparação (staging area). Ele informa quais arquivos foram modificados, quais estão na staging area e quais não estão sendo rastreados pelo Git.

Exemplo de Uso:

git status
git add

Adiciona alterações do diretório de trabalho para a área de preparação (staging area). É o passo intermediário antes de confirmar (commitar) as alterações.

Exemplos de Uso:

# Adiciona um arquivo específico
git add nome_do_arquivo.txt

# Adiciona todos os arquivos modificados e novos
git add .
git commit

Salva um "snapshot" das alterações que estão na staging area para o histórico do projeto. Cada commit tem um ID único e uma mensagem descritiva.

Exemplos de Uso:

# Abre o editor de texto para escrever uma mensagem de commit
git commit

# Commita com uma mensagem diretamente na linha de comando
git commit -m "Adiciona funcionalidade de login"
git pull

Busca e mescla (merge) as alterações de um repositório remoto para o seu branch local. É uma combinação dos comandos git fetch e git merge.

Exemplo de Uso:

# Atualiza o branch local com as alterações do branch remoto 'main'
git pull origin main
git push

Envia os commits do seu branch local para o repositório remoto correspondente. Isso atualiza o repositório remoto com as alterações que você fez localmente.

Exemplo de Uso:

# Envia os commits para o branch remoto 'main'
git push origin main
git branch

Lista, cria ou deleta branches. Branches são "linhas do tempo" independentes no seu projeto, usadas para desenvolver funcionalidades isoladamente.

Exemplos de Uso:

# Lista todos os branches locais
git branch

# Cria um novo branch chamado 'nova-feature'
git branch nova-feature

# Deleta um branch
git branch -d nome-do-branch
git checkout

Muda de um branch para outro. Também pode ser usado para restaurar arquivos do diretório de trabalho.

Exemplos de Uso:

# Muda para o branch 'nova-feature'
git checkout nova-feature

# Cria e já muda para um novo branch
git checkout -b outra-feature
git merge

Une o histórico de um branch independente ao seu branch atual. Por exemplo, para incorporar as mudanças de um branch de 'feature' no branch 'main'.

Exemplo de Uso:

# Primeiro, mude para o branch que receberá as alterações
git checkout main

# Agora, faça o merge do branch 'nova-feature' no 'main'
git merge nova-feature
git log

Exibe o histórico de commits do branch atual, mostrando o autor, data e mensagem de cada commit.

Exemplos de Uso:

# Mostra o log completo
git log

# Mostra o log de forma mais concisa, com uma linha por commit
git log --oneline

# Mostra o log com um gráfico das ramificações
git log --oneline --graph --all
git remote

Gerencia as conexões com repositórios remotos. Você pode listar, adicionar ou remover repositórios remotos.

Exemplos de Uso:

# Lista os repositórios remotos configurados (geralmente 'origin')
git remote -v

# Adiciona um novo repositório remoto
git remote add origin https://github.com/usuario/novo-repo.git
