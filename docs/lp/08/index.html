Problema 1: Código que se Repete

Imagine que você precisa exibir um cabeçalho formatado várias vezes no seu programa. Você acabaria fazendo isso:

// Início do programa
printf("**************************\n");
printf("* Sistema Escolar   *\n");
printf("**************************\n");
// ...algum código...
// Mostrar o menu
printf("**************************\n");
printf("* Sistema Escolar   *\n");
printf("**************************\n");
printf("1- Cadastrar\n2- Listar\n");
// ...outro código...

Problema: Se você precisar mudar o cabeçalho, terá que alterá-lo em todos os lugares. É ineficiente e propenso a erros.

Problema 2: A Função main Monolítica

À medida que nossos programas crescem, colocar todo o código dentro da função main a torna um "monstro": longa, confusa e difícil de entender.

int main() {
    // Bloco de código para cadastrar 3 alunos...
    // (várias linhas de scanf e printf)
    for(...) { ... }
    // Bloco de código para encontrar o aluno com maior nota...
    // (variáveis, laço for, if, etc)
    for(...) { ... }
    // Bloco de código para exibir a turma toda...
    // (outro laço for com printf)
    for(...) { ... }
}

Problema: É difícil ler, depurar (encontrar erros) e reutilizar qualquer parte desse código em outro programa.

A Solução: Funções!

Uma função é um bloco de código nomeado que realiza uma tarefa específica. Pense nela como um "mini-programa" ou uma "receita" que podemos chamar sempre que precisarmos.

Reutilização

Escrevemos o código uma vez e o chamamos quantas vezes quisermos.

Organização

Quebramos um problema grande em partes menores e mais fáceis de gerenciar.

Legibilidade

A função main fica limpa, apenas orquestrando as chamadas para outras funções.

A Anatomia de uma Função

Uma função tem 4 partes principais, cada uma com sua cor:

int somar(int a, int b) {
    int resultado = a + b;
    return resultado;
}
  • 1. Tipo de Retorno: O tipo de dado que a função "devolve" após terminar.
  • 2. Nome da Função: Um nome que descreve o que ela faz.
  • 3. Parâmetros: As "informações" que a função precisa receber para trabalhar.
  • 4. Corpo da Função: O bloco de código que executa a tarefa. A palavra-chave return envia o valor de volta e finaliza a execução da função imediatamente.

Depurador: Fluxo de Execução

Use a seta para a direita (→) para ver como o programa "pula" entre a main e a função.

Código

int somar(int a, int b) {
    int resultado = a + b;
    return resultado;
}
int main() {
    int total;
    total = somar(5, 3);
    printf("Resultado: %d\n", total);
    return 0;
}

Funções que Não Retornam Nada (void)

E se uma função só precisa fazer uma ação, mas não precisa devolver um valor? Usamos o tipo de retorno void (vazio).

// Esta função apenas imprime uma linha na tela. Não há valor para retornar.
void imprimirLinha() {
    printf("---------------------\n");
}
int main(){
    imprimirLinha();
    printf("Relatório de Vendas\n");
    imprimirLinha();
    return 0;
}

Funções void não usam a palavra-chave return.

Depurador: Passagem por Valor

A função recebe uma CÓPIA da variável. Use a seta para a direita (→) para ver que a variável original não se altera.

Código

void tentarMudar(int x) {
    x = 99;
}
int main() {
    int numero = 10;
    tentarMudar(numero);
    printf("Valor final: %d", numero);
    return 0;
}

Exemplo 1: Calculadora Simples

#include <stdio.h>
int somar(int a, int b) {
    return a + b;
}
int subtrair(int a, int b) {
    return a - b;
}
void mostrarMenu() {
    printf("1 - Somar\n");
    printf("2 - Subtrair\n");
}
int main() {
    int n1 = 10, n2 = 5;
    int resultadoSoma, resultadoSub;
    mostrarMenu();
    resultadoSoma = somar(n1, n2);
    resultadoSub = subtrair(n1, n2);
    printf("Soma: %d\n", resultadoSoma);
    printf("Subtracao: %d\n", resultadoSub);
    return 0;
}

Exemplo 2: Calculando Área

Criamos uma função específica para calcular a área de um retângulo. Isso torna o cálculo reutilizável e o main mais limpo.

#include <stdio.h>
float calcularAreaRetangulo(float base, float altura) {
    float area = base * altura;
    return area;
}
int main() {
    float baseUsuario, alturaUsuario;
    float areaFinal;
    printf("Digite a base: ");
    scanf("%f", &baseUsuario);
    printf("Digite a altura: ");
    scanf("%f", &alturaUsuario);
    areaFinal = calcularAreaRetangulo(baseUsuario, alturaUsuario);
    printf("A área do retângulo é: %.2f\n", areaFinal);
    return 0;
}

Exemplo 3: Verificador Par ou Ímpar

Esta função recebe um número e retorna 1 se ele for par ou 0 se for ímpar. O main usa o resultado para imprimir a mensagem correta.

#include <stdio.h>
int ehPar(int numero) {
    if (numero % 2 == 0) {
        return 1; // Verdadeiro (é par)
    } else {
        return 0; // Falso (é ímpar)
    }
}
int main() {
    int num;
    printf("Digite um numero: ");
    scanf("%d", &num);
    if (ehPar(num) == 1) {
        printf("O numero %d e PAR.\n", num);
    } else {
        printf("O numero %d e IMPAR.\n", num);
    }
    return 0;
}

Resumo da Aula

  • Funções são blocos de código nomeados que resolvem um problema específico, promovendo reutilização e organização.
  • Uma função é definida por seu tipo de retorno, nome e parâmetros.
  • Funções com retorno void executam uma ação, mas não devolvem um valor.
  • Por padrão, C usa passagem por valor: a função recebe uma cópia dos dados, e a variável original não é alterada.
