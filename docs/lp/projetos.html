Propostas de Projeto Final

Escolha um dos projetos abaixo para desenvolver em grupo nas próximas duas semanas.

Instruções Gerais

  • Os projetos devem ser desenvolvidos em grupos de no mínimo 3 e no máximo 5 integrantes. Grupos menores serão mesclados.
  • O controle de versão do projeto deve ser feito obrigatoriamente via Github.
  • É fundamental que o histórico de commits no Github mostre contribuições de TODOS os integrantes do grupo.
  • Todos os grupos irão apresentar seus projetos para a turma, explicando o funcionamento do programa. O tempo máximo para cada apresentação é de 15 minutos. Apresentações que excederem o tempo serão interrompidas.
  • Todos os projetos devem ser modularizados, separando a lógica em diferentes arquivos .h e .c quando apropriado.
  • Todos os projetos devem implementar um sistema de persistência de dados, salvando as informações em um arquivo de texto para que não se percam ao fechar o programa.
  • A interface do usuário deve ser um menu de opções infinito (usando while ou do-while) que se repete até que o usuário escolha a opção "Sair".
  • Os arquivos .h fornecidos são um ponto de partida. Vocês podem e devem alterar as funções, adicionando mais parâmetros se julgarem necessário para a lógica do programa.

1. Agenda de Contatos

Conceito Principal

Criar um sistema que permita ao usuário cadastrar, listar, buscar e remover contatos. Os contatos devem ser salvos em um arquivo para que a agenda seja permanente.

Funcionalidades Mínimas

  • Usar uma struct Contato (nome, telefone, email).
  • Permitir cadastrar um novo contato.
  • Listar todos os contatos cadastrados.
  • Buscar um contato pelo nome.
  • Salvar a lista de contatos em um arquivo agenda.txt.
  • Carregar os contatos do arquivo ao iniciar o programa.

Arquivo de Header Obrigatório (agenda.h)

#ifndef AGENDA_H
#define AGENDA_H

#define MAX_CONTATOS 100

typedef struct {
    char nome[50];
    char telefone[20];
    char email[50];
} Contato;

void cadastrar_contato(Contato agenda[], int *total_contatos);
void listar_contatos(Contato agenda[], int total_contatos);
void buscar_contato(Contato agenda[], int total_contatos);
void salvar_agenda(Contato agenda[], int total_contatos);
void carregar_agenda(Contato agenda[], int *total_contatos);

#endif

2. Gerenciador de Estoque de Loja

Conceito Principal

Desenvolver um programa para controlar o estoque de uma pequena loja. O sistema deve permitir adicionar produtos, atualizar a quantidade e listar os itens.

Funcionalidades Mínimas

  • Usar uma struct Produto (codigo, nome, quantidade, preco).
  • Permitir cadastrar um novo produto.
  • Listar todos os produtos do estoque.
  • Permitir a entrada (compra) e saída (venda) de produtos, atualizando a quantidade.
  • Salvar e carregar o estado do estoque de um arquivo estoque.txt.

Arquivo de Header Obrigatório (estoque.h)

#ifndef ESTOQUE_H
#define ESTOQUE_H

#define MAX_PRODUTOS 50

typedef struct {
    int codigo;
    char nome[50];
    int quantidade;
    float preco;
} Produto;

void cadastrar_produto(Produto estoque[], int *total_produtos);
void listar_produtos(Produto estoque[], int total_produtos);
void atualizar_estoque(Produto estoque[], int total_produtos);
void salvar_estoque(Produto estoque[], int total_produtos);
void carregar_estoque(Produto estoque[], int *total_produtos);

#endif

3. Sistema de Biblioteca

Conceito Principal

Criar um sistema para gerenciar o empréstimo de livros de uma biblioteca. O programa deve controlar quais livros estão disponíveis e quais estão emprestados.

Funcionalidades Mínimas

  • Usar uma struct Livro (titulo, autor, status - 0 para disponível, 1 para emprestado).
  • Permitir cadastrar novos livros.
  • Listar todos os livros, mostrando seu status.
  • Realizar o empréstimo de um livro (mudar status para 1).
  • Realizar a devolução de um livro (mudar status para 0).
  • Salvar e carregar a lista de livros de um arquivo biblioteca.txt.

Arquivo de Header Obrigatório (biblioteca.h)

#ifndef BIBLIOTECA_H
#define BIBLIOTECA_H

#define MAX_LIVROS 100

typedef struct {
    char titulo[100];
    char autor[50];
    int status; // 0 = disponível, 1 = emprestado
} Livro;

void cadastrar_livro(Livro acervo[], int *total_livros);
void listar_livros(Livro acervo[], int total_livros);
void emprestar_livro(Livro acervo[], int total_livros);
void devolver_livro(Livro acervo[], int total_livros);
void salvar_biblioteca(Livro acervo[], int total_livros);
void carregar_biblioteca(Livro acervo[], int *total_livros);

#endif

5. Sistema de Cadastro de Alunos

Conceito Principal

Desenvolver um sistema para gerenciar os registros de alunos de uma turma, incluindo suas notas e status final (aprovado/reprovado).

Funcionalidades Mínimas

  • Usar uma struct Aluno (nome, ra, nota1, nota2). A média e o status podem ser calculados.

6. Gerenciador de Tarefas (To-Do List)

Conceito Principal

Criar um programa de lista de tarefas onde o usuário pode adicionar novas tarefas, listá-las e marcá-las como concluídas.

Funcionalidades Mínimas

  • Usar uma struct Tarefa (descricao, prioridade (1-5), status - 0 para pendente, 1 para concluída).
  • Permitir adicionar uma nova tarefa.
  • Listar todas as tarefas, mostrando seu status (ex: [X] para concluída, [ ] para pendente).
  • Permitir marcar uma tarefa como concluída, buscando-a pelo seu número na lista.
  • Salvar e carregar as tarefas de um arquivo tarefas.txt.

Arquivo de Header Obrigatório (tarefas.h)

#ifndef TAREFAS_H
#define TAREFAS_H

#define MAX_TAREFAS 100

typedef struct {
    char descricao[100];
    int prioridade;
    int status; // 0 = pendente, 1 = concluída
} Tarefa;

void adicionar_tarefa(Tarefa lista[], int *total_tarefas);
void listar_tarefas(Tarefa lista[], int total_tarefas);
void marcar_concluida(Tarefa lista[], int total_tarefas);
void salvar_tarefas(Tarefa lista[], int total_tarefas);
void carregar_tarefas(Tarefa lista[], int *total_tarefas);

#endif

8. Sistema de Reserva de Hotel

Conceito Principal

Criar um sistema que simule a reserva de quartos em um pequeno hotel (10 quartos). O programa deve mostrar quais quartos estão livres e permitir a reserva.

Funcionalidades Mínimas

  • Usar um vetor de struct Quarto (numero, status - 0 para livre, 1 para ocupado, nome_hospede).
  • Iniciar o hotel com 10 quartos, todos livres.
  • Listar todos os quartos, mostrando se estão "Livre" ou "Ocupado por [Nome]".
  • Permitir reservar um quarto, pedindo o número do quarto e o nome do hóspede.
  • Permitir liberar um quarto (check-out).
  • Salvar e carregar o estado dos quartos de um arquivo hotel.txt.

Arquivo de Header Obrigatório (hotel.h)

#ifndef HOTEL_H
#define HOTEL_H

#define NUM_QUARTOS 10

typedef struct {
    int numero;
    int status; // 0 = livre, 1 = ocupado
    char nome_hospede[50];

void inicializar_hotel(Quarto hotel[]);
void listar_quartos(Quarto hotel[]);
void reservar_quarto(Quarto hotel[]);
void liberar_quarto(Quarto hotel[]);

#endif

10. Jogo de Quiz com Recorde

Conceito Principal

Um jogo de perguntas e respostas. As perguntas devem ser lidas de um arquivo, e o programa deve salvar o recorde (maior pontuação) em outro arquivo.

Funcionalidades Mínimas

  • Usar uma struct Pergunta e uma struct Jogador.
  • Ler um conjunto de pelo menos 5 perguntas de um arquivo perguntas.txt.
  • Apresentar cada pergunta ao jogador e verificar se a resposta está correta.
  • Manter uma pontuação e exibi-la no final do jogo.
  • Carregar o recorde atual (nome e pontos) do arquivo recorde.txt no início.
  • Se o jogador atual quebrar o recorde, salvar seu nome e nova pontuação no arquivo.

Arquivo de Header Obrigatório (quiz.h)

#ifndef QUIZ_H
#define QUIZ_H

#define MAX_PERGUNTAS 20

typedef struct {
    char enunciado[200];
    char opcoes[4][100];
    int resposta_correta; // 1, 2, 3 ou 4
} Pergunta;

typedef struct {
    char nome[50];
    int pontuacao;
} Jogador;

int carregar_perguntas(Pergunta quiz[], const char* nome_arquivo);
void iniciar_quiz(Pergunta quiz[], int total_perguntas);
void carregar_recorde(Jogador *recordista);
void salvar_recorde(Jogador recordista);

#endif
