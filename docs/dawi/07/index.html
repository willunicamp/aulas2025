A Magia do JavaScript no Bootstrap

Até agora, só usamos o CSS do Bootstrap. Mas como componentes como Modals e Alertas ganham vida?

A resposta está no arquivo bootstrap.bundle.min.js. Ele contém todo o código JavaScript necessário para dar interatividade aos componentes.

Preciso saber JavaScript para usar isso?

Não! Você não precisa escrever uma linha de JavaScript para usar os componentes interativos. O Bootstrap foi projetado para que você possa controlar tudo diretamente do seu HTML.

Nesta aula, vamos entender os conceitos por trás disso para que você saiba *como* a mágica acontece.

O que é o DOM? (Uma explicação simples)

Para o JavaScript funcionar, ele precisa "ver" seu HTML de uma forma organizada.

Imagine seu código HTML como uma árvore genealógica. O <html> é o ancestral principal. O <body> e o <head> são seus filhos. O <div> dentro do body é um neto, e assim por diante.

Essa "árvore" de elementos é chamada de DOM (Document Object Model). O JavaScript do Bootstrap sabe como navegar por essa árvore para encontrar, modificar, esconder ou mostrar elementos, criando a interatividade que vemos.

A Chave Mestra: Atributos data-bs-*

Como dizemos ao JavaScript do Bootstrap o que fazer, sem escrever JavaScript?

Nós usamos atributos de dados no HTML. Eles são como etiquetas especiais que o JavaScript do Bootstrap procura no DOM.

O padrão é sempre data-bs-*. Por exemplo:

<!-- Este atributo "diz" ao JS: "Quando este botão for clicado,
     procure e abra o Modal com o ID 'meuModal'" -->

<button data-bs-toggle="modal" data-bs-target="#meuModal">
  Abrir Modal
</button>

Esses atributos são as "instruções" que passamos do nosso HTML para o JavaScript do Bootstrap. Simples assim!

Alertas Dispensáveis (Dismissible)

Vamos ao primeiro exemplo prático: um alerta que o usuário pode fechar.

Precisamos de 3 coisas no nosso HTML:

  1. A classe .alert-dismissible no alerta para adicionar espaçamento para o botão de fechar.
  2. Um <button> dentro do alerta com a classe .btn-close.
  3. O atributo mágico data-bs-dismiss="alert" no botão. Esta é a instrução para o JS!
<div class="alert alert-warning alert-dismissible fade show" role="alert">
  Atenção! Seus dados serão atualizados.
  <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
</div>

Resultado:

Atenção! Seus dados serão atualizados.

Modals: Janelas Sobrepostas

Modals são janelas de diálogo que aparecem sobre o conteúdo da página, exigindo a atenção do usuário.

Quando usar um Modal?

  • Para exibir informações importantes sem sair da página.
  • Para formulários curtos (como login ou inscrição).
  • Para confirmações de ações críticas (ex: "Você tem certeza que quer excluir este item?").
  • Para exibir conteúdo maior, como imagens ou vídeos.

A Estrutura de um Modal

Um Modal é composto por duas partes principais no HTML:

1. O Gatilho (Trigger)

Geralmente um botão que o usuário clica para abrir o modal. Ele usa data-bs-toggle="modal" para se identificar e data-bs-target="#idDoModal" para "mirar" no modal certo.

<button type="button" class="btn btn-primary"
  data-bs-toggle="modal"
  data-bs-target="#exemploModal">
  Abrir Modal
</button>

2. O Conteúdo do Modal

O HTML da janela que será exibida. Ele precisa ter o id correspondente ao `target` do gatilho. Geralmente é colocado no final do <body>.

<div class="modal fade" id="exemploModal">
  <!-- Conteúdo do Modal aqui -->
</div>

Exemplo Completo de Modal

Vamos ver o código completo do conteúdo do modal e um exemplo funcional.

<!-- O código do Modal (geralmente no fim do <body>) -->
<div class="modal fade" id="exemploModal" tabindex="-1" aria-labelledby="tituloModal" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title" id="tituloModal">Título do Modal</h5>
        <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
      </div>
      <div class="modal-body">
        O conteúdo do seu modal vai aqui.
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Fechar</button>
        <button type="button" class="btn btn-primary">Salvar mudanças</button>
      </div>
    </div>
  </div>
</div>

Formulários com Bootstrap

Formulários são uma das partes mais importantes e complexas de qualquer aplicação web. Bootstrap nos dá um conjunto poderoso de classes para estilizá-los de forma rápida e consistente.

Controles de Formulário Básicos

A classe mais fundamental é a .form-control, aplicada a inputs de texto, email, etc. Envolvemos cada campo em um <div> com .mb-3 para espaçamento.

<div class="mb-3">
  <label for="exemploInput" class="form-label">Endereço de Email</label>
  <input type="email" class="form-control" id="exemploInput" placeholder="nome@exemplo.com">
</div>
<div class="mb-3">
  <label for="exemploTextarea" class="form-label">Sua Mensagem</label>
  <textarea class="form-control" id="exemploTextarea" rows="3"></textarea>
</div>

Validação Real (com JavaScript)

Para uma experiência de usuário melhor, a validação deve acontecer após o usuário tentar enviar o formulário.

Para isso, usamos a classe .needs-validation no formulário e o atributo novalidate. Depois, adicionamos um pequeno script (que você pode copiar e colar) para ativar a lógica.

<!-- No seu formulário -->
<form class="row g-3 needs-validation" novalidate>
  <div class="col-md-6">
    <label for="validationCustom01" class="form-label">Nome</label>
    <input type="text" class="form-control" id="validationCustom01" required>
    <div class="valid-feedback">
      Tudo certo!
    </div>
    <div class="invalid-feedback">
      Por favor, insira seu nome.
    </div>
  </div>
  <!-- ... outros campos ... -->
</form>

<!-- No seu <script> no final da página -->
<script>
  (() => {
    'use strict'
    const forms = document.querySelectorAll('.needs-validation')
    Array.from(forms).forEach(form => {
      form.addEventListener('submit', event => {
        if (!form.checkValidity()) {
          event.preventDefault()
          event.stopPropagation()
        }
        form.classList.add('was-validated')
      }, false)
    })
  })()
</script>
O que esse script faz? Ele encontra todos os formulários com a classe .needs-validation. Quando o usuário clica em "enviar", o script verifica se tudo está preenchido corretamente. Se não estiver, ele impede o envio e adiciona a classe .was-validated para mostrar os erros.

O Coração da Navegação: Navbars

A Navbar é um dos componentes mais essenciais de um site, sendo totalmente responsiva. Ela "colapsa" em telas menores, transformando os links em um menu "hambúrguer".

<nav class="navbar navbar-expand-lg bg-dark navbar-dark">
  <div class="container-fluid">
    <a class="navbar-brand" href="#">Meu Site</a>
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#conteudoNavbar">
      <span class="navbar-toggler-icon"></span>
    </button>
    <div class="collapse navbar-collapse" id="conteudoNavbar">
      <ul class="navbar-nav me-auto mb-2 mb-lg-0">
        <li class="nav-item">
          <a class="nav-link active" href="#">Início</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#">Sobre</a>
        </li>
      </ul>
    </div>
  </div>
</nav>

Resultado:

Meu Site
  • Início
  • Sobre

Anatomia de uma Navbar

  • .navbar-expand-lg: Diz em qual breakpoint (lg = large) a navbar deve expandir. Abaixo disso, ela colapsa.
  • .navbar-brand: Para o logo ou nome do site.
  • .navbar-toggler: O botão "hambúrguer" que aparece em telas pequenas.
  • data-bs-target="#id": Aponta para o ID do conteúdo que deve ser expandido/colapsado.
  • .collapse .navbar-collapse: O contêiner dos links que será escondido/mostrado.

Ativando Componentes via JS (O único passo!)

Por razões de performance, Tooltips e Popovers são os únicos componentes que precisam ser "ligados" manualmente com um pequeno trecho de JavaScript.

// Este código procura todos os elementos com data-bs-toggle="tooltip"
// e os inicializa.
const tooltipTriggerList = document.querySelectorAll('[data-bs-toggle="tooltip"]');
const tooltipList = [...tooltipTriggerList].map(tooltipTriggerEl => new bootstrap.Tooltip(tooltipTriggerEl));

// Este faz o mesmo para os Popovers.
const popoverTriggerList = document.querySelectorAll('[data-bs-toggle="popover"]');
const popoverList = [...popoverTriggerList].map(popoverTriggerEl => new bootstrap.Popover(popoverTriggerEl));
Importante: Este código deve ser colocado dentro da sua tag <script> para que funcione.

Tooltips e Popovers: O Código

Tooltips são para dicas rápidas. Popovers permitem mais conteúdo, com título e corpo.

Tooltip
<button type="button" class="btn btn-secondary"
  data-bs-toggle="tooltip"
  data-bs-placement="top"
  data-bs-title="Isso é um tooltip!">
  Passe o mouse aqui
</button>
Popover
<button type="button" class="btn btn-danger"
  data-bs-toggle="popover"
  data-bs-title="Título do Popover"
  data-bs-content="E aqui está o conteúdo.">
  Clique aqui
</button>

Conteúdo Retrátil: Accordion

O componente Accordion (ou "sanfona") é perfeito para seções de FAQ (Perguntas Frequentes). Ele permite expandir e colapsar blocos de conteúdo.

<div class="accordion" id="accordionExemplo">
  <div class="accordion-item">
    <h2 class="accordion-header">
      <button class="accordion-button" ...>
        Pergunta #1
      </button>
    </h2>
    <div id="collapseUm" class="accordion-collapse collapse show" data-bs-parent="#accordionExemplo">
      <div class="accordion-body">
        Resposta #1.
      </div>
    </div>
  </div>
</div>

O atributo data-bs-parent="#idDoAccordion" garante que apenas um item do accordion fique aberto por vez. Se removê-lo, o usuário poderá abrir vários.
