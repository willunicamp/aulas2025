Passo 2: O Estilo Base (Mobile First)

Agora, vamos criar o arquivo style.css. Começaremos pelo visual mobile, que é o mais simples. Por padrão, os elementos <section> ficarão um embaixo do outro.

2.1 - Estilos Gerais

Adicione estilos básicos para as seções e para os componentes internos.

/* Mobile First - Estilos Base */
section {
  background-color: #0f3460;
  border: 1px solid #e94560;
  border-radius: 8px;
  padding: 1rem;
  margin-bottom: 1rem;
}
h2 {
  margin-top: 0;
  border-bottom: 2px solid #e94560;
  padding-bottom: 0.5rem;
  margin-bottom: 1rem;
}
img {
  max-width: 100%;
  height: auto;
  border-radius: 4px;
}

Resultado esperado: Todas as seções agora têm um fundo, borda e espaçamento, empilhadas verticalmente.

2.2 - Usando Flexbox nos Componentes

Vamos usar Flexbox para organizar o conteúdo DENTRO das seções de Perfil e Inventário.

/* Perfil */
.perfil {
  display: flex;
  align-items: center; /* Alinha a imagem e os status verticalmente */
  gap: 1rem;
}
.perfil img {
  border-radius: 50%; /* Deixa o avatar redondo */
}

/* Inventário */
.itens {
  display: flex;
  flex-wrap: wrap; /* Permite que os itens quebrem a linha */
  gap: 0.5rem;
}
.item {
  background-color: #1a1a2e;
  padding: 0.5rem 1rem;
  border-radius: 4px;
  text-align: center;
  flex-grow: 1; /* Faz os itens crescerem para preencher o espaço */
}

Resultado esperado: No perfil, o avatar e os status agora estão lado a lado. No inventário, os itens formam uma grade flexível que se ajusta à largura.

Passo 3: O Layout de Tablet (Media Query + Grid)

É hora da primeira grande transformação! Para telas com 768px ou mais, vamos usar CSS Grid para criar um layout de 2 colunas.

@media (min-width: 768px) {
  .painel {
    display: grid;
    gap: 1rem;
    grid-template-columns: 1fr 1fr; /* Duas colunas de tamanho igual */
    grid-template-areas:
      "perfil inventario"
      "missoes inventario"
      "mapa mapa";
  }

  /* Associamos cada seção à sua área */
  .perfil { grid-area: perfil; }
  .inventario { grid-area: inventario; }
  .missoes { grid-area: missoes; }
  .mapa { grid-area: mapa; }

  /* Ajuste para remover a margem que não é mais necessária */
  section {
    margin-bottom: 0;
  }
}

Resultado esperado: Ao aumentar a janela do navegador para mais de 768px, o layout deve "saltar" da pilha vertical para o layout de 2 colunas que definimos no grid-template-areas.

Passo 4: O Layout de Desktop (A Evolução Final)

Para telas grandes (1024px ou mais), vamos criar um layout ainda mais sofisticado, com 4 colunas, dando mais destaque às missões.

@media (min-width: 1024px) {
  .painel {
    /* Apenas reescrevemos o "mapa" e as colunas */
    grid-template-columns: 250px 1fr 1fr 300px; /* 4 colunas com tamanhos diferentes */
    grid-template-areas:
      "perfil missoes missoes mapa"
      "inventario missoes missoes mapa";
  }
}

Resultado esperado: Ao passar dos 1024px de largura, o layout se reorganiza novamente para a versão final de 4 colunas, mostrando o poder de redefinir o grid-template-areas em diferentes breakpoints.
