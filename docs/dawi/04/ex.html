Criar a vitrine da loja, com jogos em destaque e uma galeria de produtos que se adapta a qualquer tela.

Código de Partida (index.html)

<!DOCTYPE html>
<html>
<head>
  <title>Loja de Games</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header>Cabeçalho/Menu</header>
  <main>
    <section class="destaques">
      <div class="destaque-principal"><img src="destaque-01.png"></div>
      <div class="destaques-secundarios">
        <div class="destaque-secundario"><img src="destaque-02.png"></div>
        <div class="destaque-secundario"><img src="destaque-03.png"></div>
      </div>
    </section>
    <section class="galeria-jogos">
      <!-- Repita este card 10 vezes -->
      <div class="card-jogo">
        <img src="jogo-01.png">
        <div class="info-card">
          <p class="descricao">Descrição breve do jogo...</p>
          <div class="botoes-card">
            <button class="btn-comprar">Comprar</button>
            <button class="btn-carrinho">Carrinho</button>
          </div>
        </div>
      </div>
    </section>
  </main>
</body>
</html>

Requisitos (CSS)

  1. Estilo Base (Mobile First): O .destaques-secundarios deve estar escondido (display: none). A .galeria-jogos deve ser um contêiner flex com flex-wrap: wrap e justify-content: center.
  2. Card de Jogo (Mobile): A .descricao dentro do card deve estar escondida. O .botoes-card deve ser um contêiner flex com flex-direction: column.
  3. Media Query (Tablet ≥768px): Crie uma media query. A galeria deve agora comportar mais itens por linha. O .card-jogo deve se tornar um contêiner flex (display: flex). A .descricao deve reaparecer (display: block).
  4. Card de Jogo (Tablet/Desktop): O .info-card deve ser um contêiner flex com flex-direction: column. A .descricao deve ter flex-grow: 1 para empurrar os botões para baixo. O .botoes-card deve ter flex-direction: row.
  5. Media Query (Desktop ≥1024px): Crie outra media query. O .destaques deve se tornar um contêiner flex. O .destaques-secundarios deve reaparecer (display: flex) e ter flex-direction: column.
  6. Layout Desktop (Destaques): O .destaque-principal deve ter flex-grow: 1 para ocupar o espaço. Dê um tamanho base (flex-basis) para os .destaques-secundarios.

Parte 2: A Página de Detalhes do Jogo

Criar uma página de produto com 3 layouts diferentes para mobile, tablet e desktop, usando `order` para reorganizar o conteúdo.

Imagem

Código de Partida (detalhes.html)

<!DOCTYPE html>
<html>
<head>
  <title>Detalhes do Jogo</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <div class="detalhes-container">
    <div class="imagem-jogo"><img src="jogo-grande-01.png"></div>
    <div class="info-jogo">Informações e descrição...</div>
    <div class="compra-jogo">Preço e botão de comprar...</div>
  </div>
</body>
</html>

Requisitos (CSS)

  1. Estilo Base (Mobile): O .detalhes-container deve ser `flex` com `flex-direction: column`. Use a propriedade `order` para que a caixa de compra (.compra-jogo) apareça antes das informações (.info-jogo).
  2. Media Query (Tablet ≥768px): O .detalhes-container deve ter `flex-wrap: wrap` e `flex-direction: row`. A imagem (.imagem-jogo) deve ocupar 100% da largura (flex-basis: 100%). As seções de info e compra devem voltar à sua ordem natural (`order: 0`).
  3. Media Query (Desktop ≥1024px): O .imagem-jogo não deve mais ocupar 100%. Dê a ele e à caixa de compra tamanhos base fixos (flex-basis) e faça a seção de informações (.info-jogo) crescer para ocupar o espaço restante (flex-grow: 1).
