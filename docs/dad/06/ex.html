Exercício: Implementando o UPDATE

Objetivo: Aplicar os conceitos de UPDATE para permitir a edição de Raças e Classes existentes.

Instruções Gerais

  • Este exercício deve ser implementado para as telas de Raças e Classes. Os passos abaixo descrevem a lógica para um formulário, que deve ser replicada para o outro.
  • O objetivo é refatorar os formulários, que hoje apenas inserem dados, para que também possam carregar, exibir e atualizar dados existentes.
  • Utilize os conceitos de variável de controle (flag), evento SelectedIndexChanged e o comando SQL UPDATE vistos na Aula 06.

Passo 1: Adapte a Interface

Adicione um controle ListBox ao seu formulário de Raças (e Classes). Este ListBox será usado para exibir todos os registros já cadastrados no banco de dados.

Passo 2: Implemente a Leitura (Read)

Crie um método CarregarRacas() (e CarregarClasses()). Este método deve:

  • Limpar o ListBox.
  • Executar um SELECT para buscar todos os registros da tabela.
  • Popular o ListBox com os dados, utilizando uma classe auxiliar (ex: RacaListBoxItem) e sobrescrevendo o método ToString().
  • Chame este método no evento Load do formulário.

Passo 3: Habilite a Edição

Implemente o evento SelectedIndexChanged do ListBox. Dentro deste evento, você deve:

  • Obter o objeto selecionado na lista (ex: RacaListBoxItem).
  • Usar os dados do objeto para preencher os campos do formulário (TextBox, NumericUpDown, etc.).
  • Armazenar o ID do item selecionado em uma variável de controle (ex: idRacaSelecionada).

Passo 4: Botão Inteligente

Refatore o seu botão btnSalvar_Click. Ele deve agora verificar a variável de controle:

  • Se o ID for 0, ele deve chamar um método para inserir um novo registro (a lógica que você já tinha).
  • Se o ID for maior que 0, ele deve chamar um novo método para atualizar o registro existente.

Passo 5: Implemente o UPDATE

Crie o método para atualizar (ex: AtualizarRaca()). Este método deve:

  • Montar uma string com o comando SQL UPDATE, utilizando a cláusula WHERE id = @id.
  • Adicionar todos os valores dos campos como parâmetros, incluindo o ID que está na sua variável de controle.
  • Executar o comando com ExecuteNonQuery().
  • Após a execução, limpar os campos, recarregar o ListBox e exibir uma MessageBox de sucesso.

Dica de Ouro (Extra)

Adicione um botão "Novo" ou "Limpar" ao seu formulário. A única função dele será chamar um método LimparCampos(), que limpa todos os TextBoxes e, o mais importante, reseta sua variável de controle (idRacaSelecionada = 0;). Isso permite que o usuário cancele uma edição e volte ao modo de criação.
