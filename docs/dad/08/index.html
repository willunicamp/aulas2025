A Origem dos Heróis

Vamos solidificar nossa base em Orientação a Objetos, aprendendo a criar hierarquias de classes com Herança, Sobrescrita e Sobrecarga.

Anatomia de uma Classe (Interativo)

Uma Classe é um molde que agrupa Atributos e Métodos. Pressione a seta para a direita para destacar cada parte.

public class SuperHeroi
{
    // Atributos (Propriedades)
    public string PoderEspecial { get; set; }
    public int Nivel { get; set; }

    // Métodos (Ações)
    public virtual string UsaPoderEspecial()
    {
        return $"Usando o poder: {PoderEspecial}!";
    }
}

1. Classe: O molde completo, nosso projeto de SuperHeroi.

2. Atributos: As características, ou "o que ele tem". Ex: PoderEspecial, Nivel.

3. Métodos: As ações, ou "o que ele faz". Ex: UsaPoderEspecial().

Método virtual: A palavra virtual é um "convite". Ela sinaliza que qualquer classe filha que herdar de SuperHeroi tem a permissão para modificar (sobrescrever) este método, se precisar.
Níveis de Acesso: public significa que qualquer um pode ver e usar. Outros níveis são private (só a própria classe vê), protected (a classe e suas filhas veem) e internal (qualquer um no mesmo projeto vê).

Propriedades: O Jeito Longo vs. O Jeito Curto

A sintaxe { get; set; } é um atalho do C#. Por trás dos panos, o compilador cria a versão "longa" para nós.

A Forma "Longa" (Clássica)

// Campo privado para guardar o valor
private string _poderEspecial;

// Propriedade pública para controlar o acesso
public string PoderEspecial
{
    get { return _poderEspecial; }
    set { _poderEspecial = value; }
}

A Forma "Curta" (Autoimplementada)

// O compilador cria o campo privado
// e o get/set para nós!
public string PoderEspecial { get; set; }
Pense no { get; set; } como uma promessa: começamos com ele pela simplicidade, com a promessa de que, se precisarmos de validação no futuro (como não permitir um nível negativo), nós o expandiremos para a "forma longa" para adicionar essa lógica.

Usando o Molde: Criando Objetos

Com a classe (molde) pronta, podemos criar Objetos (instâncias) e interagir com eles. Pressione a seta para a direita para ver passo a passo.

Instanciando e Usando um Herói

// 1. Instancia um novo objeto da classe SuperHeroi
SuperHeroi meuHeroi = new SuperHeroi();

// 2. Define os valores dos seus atributos
meuHeroi.PoderEspecial = "Super Força";
meuHeroi.Nivel = 10;

// 3. Chama seus métodos para executar ações
string resultado = meuHeroi.UsaPoderEspecial();

// A variável 'resultado' agora contém "Usando o poder: Super Força!"

O Poder da Herança: Evitando Repetição

Herança permite que uma classe filha adquira os atributos e métodos de uma classe pai. É o princípio do "É UM". Por exemplo: "O SuperComum **É UM** SuperHeroi".

Classe Pai (Base)

SuperHeroi é a nossa classe pai. Ela define tudo que é comum a todos os heróis.

Classes Filhas (Derivadas)

HomemOvoFrito, SuperComum, etc, são classes filhas. Elas herdam tudo de SuperHeroi e podem adicionar suas próprias características.

Herança Simples: O HomemOvoFrito

O HomemOvoFrito é um super-herói, então ele herda da classe SuperHeroi. Ele não tem nada de novo, então sua classe é bem simples.

public class HomemOvoFrito : SuperHeroi
{
    // Não precisa de nada aqui!
    // Ele já herdou todas as
    // propriedades e métodos
    // de SuperHeroi.
}

Herança com Adição: O SuperComum

O SuperComum também herda de SuperHeroi, mas ele tem uma característica a mais: um ponto fraco. Podemos adicionar novas propriedades e métodos na classe filha.

public class SuperComum : SuperHeroi
{
    // Nova propriedade
    public string PontoFraco { get; set; }

    // Novo método
    public string AtivaFraqueza()
    {
        return $"Ah, não! {PontoFraco}!";
    }
}

Sobrescrita: O Caso da MulherBateria

A Sobrescrita (override) permite que uma classe filha modifique um comportamento herdado. Para isso, o método na classe pai deve ser marcado como virtual.

public class MulherBateria : SuperHeroi
{
    public int Energia { get; set; }

    public override string UsaPoderEspecial()
    {
        if (Energia > 10) {
             Energia -= 10;
             return $"BZZZT! Energia: {Energia}%";
        } else {
             return "Energia baixa demais!";
        }
    }
}

Sobrecarga: O Caso do HomemPum

A Sobrecarga (overload) é ter vários métodos com o mesmo nome, mas com parâmetros diferentes.

public class HomemPum : SuperHeroi
{
    // Já herdou: UsaPoderEspecial()

    // SOBRECARGA: mesmo nome,
    // mas com parâmetro.
    public string UsaPoderEspecial(string arma)
    {
        if (arma == "fósforo")
        {
            return "PFFFF... BOOM!";
        }
        return "Pffff...";
    }
}
